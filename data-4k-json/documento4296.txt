Test Driven Development (TDD) é uma prática muito utilizada no processo de desenvolvimento de sistemas computacionais, que aumenta a produtividade do processo a partir de testes já consolidados. Alternativas:
A. criar um teste que falhe – fazer o código funcionar – eliminar redundâncias.
B. escrever um trecho de código – validar o código – encapsular o código.
C. identificar erros no código – validar o código – testar com várias iterações.
D. realizar testes no código – avaliar o código – implementar melhorias no código.
